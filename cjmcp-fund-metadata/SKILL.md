# cjmcp-fund-metadata 基金数据查询规范
## 描述
本技能用于规范智能体对 cjmcp-fund-metadata MCP服务的调用行为，核心规则如下：
1.  接收到匹配场景的查询请求时，**优先校验当前环境的MCP可用性**，无MCP权限/未配置/未启用时，第一时间告知用户具体的启用操作步骤；
2.  仅可通过指定的cjmcp-fund-metadata MCP服务接口获取数据，禁止使用任何其他数据源编造内容；
3.  MCP调用异常时，自动重试最多3次，重试全部失败后明确告知用户错误原因与排查方向；
4.  有效数据输出需遵循专业、清晰的结构化格式规范，保证可读性与美观度。
MCP服务配置信息：cjmcp-fund-metadata（服务地址：https://mcp.cjhxfund.com/mcp-servers/cjmcp-fund-metadata/sse）

## 使用场景
当用户发起的查询请求，涉及 cjmcp-fund-metadata MCP服务提供的接口覆盖业务范围时，必须触发本技能，具体场景包括但不限于：
1. Wind兼容代码查询、证券代码映射与跨系统转换相关需求；
2. 公募基金业绩比较基准、基金经理、申赎状态、净值、业绩表现、基本资料、证券关联关系等相关数据查询需求。

## 指令
1.  **MCP环境前置校验（最高优先级，必须最先执行）**
    接收到匹配场景的用户请求后，必须按以下顺序优先校验MCP环境可用性，校验不通过时，立即停止后续操作，按对应规则给用户发送启用指引，禁止编造任何数据：
    - 第一步：校验当前智能体是否拥有MCP调用权限。若当前为普通Chat智能体（无MCP权限），必须告知用户切换智能体的操作步骤；
    - 第二步：校验当前环境是否已配置「cjmcp-fund-metadata」MCP服务。若未配置，必须告知用户完整的配置步骤与服务地址；
    - 第三步：校验已配置的「cjmcp-fund-metadata」MCP服务是否已启用。若配置了但开关未打开，必须告知用户启用操作路径；
    - 第四步：以上校验全部通过后，方可执行后续的接口调用、重试、数据返回流程。

2.  **数据源唯一限定**
    对于匹配范围内的所有查询请求，**仅可通过指定的 cjmcp-fund-metadata MCP 服务获取数据**，禁止使用任何其他数据源、内置知识库、历史对话数据来编造或补充查询结果。

3.  **接口调用规范**
    必须根据用户的查询需求，精准匹配并调用 MCP 服务提供的以下可用接口，禁止调用清单外的接口，可用接口清单如下：
    - getWindCustomCode：Wind兼容代码与证券代码映射查询
    - getChinaMutualFundBenchmark：基金业绩比较基准信息查询
    - getChinaMutualFundManager：基金经理信息查询
    - getFundSuspendPchRedm：基金暂停申购赎回信息查询
    - getChinaMutualFundPerformance：基金业绩表现与风险指标查询
    - getChinaMutualFundDescription：基金基本资料信息查询
    - getChinaMutualFundNav：基金净值核心指标查询
    - getRelatedSecuritiesCode：证券关联关系信息查询

4.  **MCP 调用重试机制（强制）**
    当 MCP 服务调用出现异常（网络超时、连接失败、服务不可用、接口报错）时：
    - 自动进行**重试调用**，**最多重试 3 次**；
    - 每次重试间隔极短，不向用户展示重试过程；
    - 若 3 次重试全部失败，立即停止执行并按异常规则返回错误与排查指引；
    - 若重试过程中任意一次调用成功，立即停止重试，正常返回 MCP 数据。

5.  **MCP不可用场景 通知规范（强制）**
    不同场景必须使用对应的固定话术通知用户，确保指引清晰、可操作，禁止模糊表述：
    - 场景1：普通Chat智能体无MCP权限
      通知话术：
      > ⚠️ 当前环境无MCP调用权限
      > 你当前使用的「Chat」智能体不支持MCP服务调用，无法为你查询基金相关数据。
      > 操作指引：请点击聊天框右上角的「智能体」下拉菜单，选择「Builder with MCP」智能体后，重新发起查询请求。

    - 场景2：未配置cjmcp-fund-metadata MCP服务
      通知话术：
      > ⚠️ 未找到对应MCP服务配置
      > 当前环境未配置「cjmcp-fund-metadata」MCP服务，无法为你查询基金相关数据。
      > 操作指引：请前往 TRAE 「设置 → MCP」，配置以下MCP服务并启用后，重新发起查询请求：

    - 场景3：已配置MCP但未启用
      通知话术：
      > ⚠️ MCP服务未启用
      > 当前环境已配置「cjmcp-fund-metadata」MCP服务，但未启用，无法为你查询基金相关数据。
      > 操作指引：请前往 TRAE 「设置 → MCP」，找到「cjmcp-fund-metadata」服务，打开启用开关后，重新发起查询请求。

6.  **异常处理强制规则**
    当出现以下任意一种情况时，必须立即停止执行，按对应规则告知用户，绝对禁止编造任何虚假数据、兜底内容或推测性信息：
    - 前置校验不通过（无权限、未配置、未启用），按规范发送对应指引；
    - MCP 服务 3 次重试后仍不可用、返回报错、无返回数据；
    - 用户的查询需求无对应可用接口匹配。

7.  **结果输出核心原则**
    仅可将 MCP 服务接口返回的原始有效数据进行整理输出，**绝对优先级：数据真实性 > 格式规范性 > 美观度**。不得对数据进行篡改、补充不存在的字段，不得添加任何非 MCP 服务返回的投资建议、走势分析、主观评价等编造内容。

8.  **输出格式美化规范（强制）**
    严格遵循Markdown格式排版，核心指标加粗突出，多条同维度数据用表格展示，所有有效数据末尾统一标注「数据来源：cjmcp-fund-metadata MCP 服务」，禁止大段无格式文本堆砌。
